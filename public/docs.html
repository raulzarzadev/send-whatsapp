Introducción

Bienvenido a la documentación de la API de WhatsApp Multi-Instance. Esta API te permite gestionar múltiples instancias de WhatsApp simultáneamente y enviar mensajes de forma programática.

Características Principales

  • Gestión de múltiples sesiones de WhatsApp
  • Generación y lectura de códigos QR
  • Envío de mensajes por sesión
  • Persistencia de sesiones
  • Autenticación por API Key
  • Rate limiting para seguridad

URL Base

https://tu-dominio.com/api

Formato

Content-Type: application/json

Autenticación

Todas las solicitudes a la API requieren autenticación mediante una API Key en el header x-api-key.

Seguridad

Configura tu API Key en las variables de entorno (API_KEY) y mantenla segura. La API implementa rate limiting de 100 requests por minuto por IP.

Ejemplo de Header:

x-api-key: tu-api-key-secreta-aqui

Ejemplo con cURL:

curl -X GET https://tu-dominio.com/api/health \
  -H "x-api-key: tu-api-key-secreta-aqui"

Endpoints

GET /health

Verifica el estado de la API.

Respuesta Exitosa (200):

{
  "success": true,
  "message": "WhatsApp API is running",
  "timestamp": "2025-10-02T10:30:00.000Z"
}
POST /sessions

Crea una nueva sesión de WhatsApp.

Parámetros del Body:

Campo Tipo Requerido Descripción
clientId string Sí Identificador del cliente
sessionId string No ID personalizado (auto-generado si se omite)

Ejemplo de Request:

{
  "clientId": "restaurante-xyz",
  "sessionId": "session-001"
}

Respuesta Exitosa (201):

{
  "success": true,
  "data": {
    "sessionId": "session-001",
    "clientId": "restaurante-xyz",
    "status": "connecting",
    "qrCode": null,
    "phone": null,
    "createdAt": "2025-10-02T10:30:00.000Z",
    "lastActivity": "2025-10-02T10:30:00.000Z"
  }
}
GET /sessions

Obtiene todas las sesiones o filtra por clientId.

Query Parameters (Opcional):

Parámetro Tipo Descripción
clientId string Filtrar sesiones por cliente

Respuesta Exitosa (200):

{
  "success": true,
  "data": [
    {
      "sessionId": "session-001",
      "clientId": "restaurante-xyz",
      "status": "connected",
      "qrCode": null,
      "phone": "521234567890",
      "createdAt": "2025-10-02T10:30:00.000Z",
      "lastActivity": "2025-10-02T10:35:00.000Z"
    }
  ],
  "count": 1
}
GET /sessions/:sessionId

Obtiene información detallada de una sesión específica.

Parámetros de URL:

Parámetro Descripción
sessionId ID de la sesión a consultar

Respuesta Exitosa (200):

{
  "success": true,
  "data": {
    "sessionId": "session-001",
    "clientId": "restaurante-xyz",
    "status": "connected",
    "qrCode": null,
    "phone": "521234567890",
    "createdAt": "2025-10-02T10:30:00.000Z",
    "lastActivity": "2025-10-02T10:35:00.000Z"
  }
}
GET /sessions/:sessionId/qr

Obtiene el código QR de una sesión para escanear con WhatsApp.

El código QR solo está disponible cuando el estado es qr o connecting.

Respuesta Exitosa (200):

{
  "success": true,
  "data": {
    "qrCode": "data:image/png;base64,iVBORw0KGgoAAAANS...",
    "sessionId": "session-001",
    "status": "qr"
  }
}
DELETE /sessions/:sessionId

Elimina una sesión y limpia todos sus datos asociados.

Esta acción es irreversible. La sesión se desconectará y todos los datos se eliminarán permanentemente.

Respuesta Exitosa (200):

{
  "success": true,
  "message": "Session deleted successfully"
}
POST /messages/send

Envía un mensaje de WhatsApp a través de una sesión conectada.

Parámetros del Body:

Campo Tipo Requerido Descripción
sessionId string Sí ID de la sesión conectada
to string Sí Número con código de país (ej: 521234567890)
message string Sí Contenido del mensaje

Ejemplo de Request:

{
  "sessionId": "session-001",
  "to": "521234567890",
  "message": "¡Hola! Este es un mensaje de prueba."
}

Respuesta Exitosa (200):

{
  "success": true,
  "message": "Message sent successfully",
  "data": {
    "sessionId": "session-001",
    "to": "521234567890",
    "sentAt": "2025-10-02T10:35:00.000Z"
  }
}

Códigos de Error

400 - Bad Request

Solicitud malformada o parámetros faltantes.

{
  "success": false,
  "error": "clientId is required"
}

401 - Unauthorized

API Key inválida o faltante.

{
  "success": false,
  "error": "Unauthorized: Invalid or missing API key"
}

404 - Not Found

Recurso no encontrado.

{
  "success": false,
  "error": "Session not found"
}

429 - Too Many Requests

Límite de rate limiting excedido (100 req/min).

{
  "success": false,
  "error": "Too many requests. Please try again later."
}

500 - Internal Server Error

Error interno del servidor.

{
  "success": false,
  "error": "Failed to send message"
}

Ejemplos de Uso

Crear una sesión:

curl -X POST https://tu-dominio.com/api/sessions \
  -H "Content-Type: application/json" \
  -H "x-api-key: tu-api-key-secreta" \
  -d '{
    "clientId": "restaurante-xyz",
    "sessionId": "session-001"
  }'

Enviar un mensaje:

curl -X POST https://tu-dominio.com/api/messages/send \
  -H "Content-Type: application/json" \
  -H "x-api-key: tu-api-key-secreta" \
  -d '{
    "sessionId": "session-001",
    "to": "521234567890",
    "message": "¡Hola! Este es un mensaje de prueba."
  }'

Obtener código QR:

curl -X GET https://tu-dominio.com/api/sessions/session-001/qr \
  -H "x-api-key: tu-api-key-secreta"

Crear una sesión:

const apiKey = 'tu-api-key-secreta';
const baseUrl = 'https://tu-dominio.com/api';

fetch(`${baseUrl}/sessions`, {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'x-api-key': apiKey
  },
  body: JSON.stringify({
    clientId: 'restaurante-xyz',
    sessionId: 'session-001'
  })
})
.then(response => response.json())
.then(data => console.log('Sesión creada:', data))
.catch(error => console.error('Error:', error));

Enviar un mensaje:

fetch(`${baseUrl}/messages/send`, {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'x-api-key': apiKey
  },
  body: JSON.stringify({
    sessionId: 'session-001',
    to: '521234567890',
    message: '¡Hola! Este es un mensaje de prueba.'
  })
})
.then(response => response.json())
.then(data => console.log('Mensaje enviado:', data))
.catch(error => console.error('Error:', error));

Crear una sesión:

import requests

api_key = 'tu-api-key-secreta'
base_url = 'https://tu-dominio.com/api'

headers = {
    'Content-Type': 'application/json',
    'x-api-key': api_key
}

data = {
    'clientId': 'restaurante-xyz',
    'sessionId': 'session-001'
}

response = requests.post(f'{base_url}/sessions', 
                        headers=headers, 
                        json=data)

print('Sesión creada:', response.json())

Enviar un mensaje:

data = {
    'sessionId': 'session-001',
    'to': '521234567890',
    'message': '¡Hola! Este es un mensaje de prueba.'
}

response = requests.post(f'{base_url}/messages/send',
                        headers=headers,
                        json=data)

print('Mensaje enviado:', response.json())

Crear una sesión (con axios):

const axios = require('axios');

const apiKey = 'tu-api-key-secreta';
const baseUrl = 'https://tu-dominio.com/api';

async function createSession() {
  try {
    const response = await axios.post(`${baseUrl}/sessions`, {
      clientId: 'restaurante-xyz',
      sessionId: 'session-001'
    }, {
      headers: {
        'Content-Type': 'application/json',
        'x-api-key': apiKey
      }
    });
    
    console.log('Sesión creada:', response.data);
  } catch (error) {
    console.error('Error:', error.response?.data || error.message);
  }
}

createSession();

Enviar un mensaje:

async function sendMessage() {
  try {
    const response = await axios.post(`${baseUrl}/messages/send`, {
      sessionId: 'session-001',
      to: '521234567890',
      message: '¡Hola! Este es un mensaje de prueba.'
    }, {
      headers: {
        'Content-Type': 'application/json',
        'x-api-key': apiKey
      }
    });
    
    console.log('Mensaje enviado:', response.data);
  } catch (error) {
    console.error('Error:', error.response?.data || error.message);
  }
}

sendMessage();
